# Peregrine API v1 — quick reference

The full OpenAPI 3.1 spec is auto-generated by Scramble and served at :

- **Swagger UI** : `https://<your-peregrine>/docs`
- **OpenAPI JSON** : `https://<your-peregrine>/docs/api.json`

## Auth

All endpoints except `/health` require :

```
Authorization: Bearer psk_<env>_<48 hex>
```

Generated via Filament admin (`/admin/shops` → action "Generate API
key"). The plaintext token is shown ONCE — store it immediately.

Per-route ability gating (e.g. `configurations:read`, `webhooks:write`)
is enforced via the middleware ; missing scope → 403 with
`error.code = insufficient_scope`.

## Response shape

Success :
```json
{ "data": ..., "meta": { ... } }
```

Error :
```json
{ "error": { "code": "configuration_not_found", "message": "...", "details": null } }
```

## Pagination

Cursor-based via Laravel's `cursorPaginate(50)`. Reuse the opaque
`meta.next_cursor` value verbatim as `?cursor=…`.

## Rate limits

(Configurable per deployment.) Defaults : 60 req/min/key + 300 req/min/IP.
HTTP 429 with `Retry-After` header on overflow.

## Idempotency (POST endpoints)

POST endpoints accept an `Idempotency-Key` header. Same key reuse =
same response replayed (24h TTL, scoped per shop).

## Error codes

| Code | When |
|---|---|
| `missing_bearer_token` | No `Authorization: Bearer` header |
| `invalid_api_key` | Token doesn't match any active key |
| `api_key_revoked_or_expired` | Key has `revoked_at` or `expires_at` past |
| `shop_suspended` | Shop status is `suspended` |
| `insufficient_scope` | Key lacks the required ability |
| `configuration_not_found` | Out of scope or non-existent |
| `order_not_found` | No `external_order_id` match for this shop |
| `webhook_endpoint_not_found` | Endpoint not owned by this shop |
| `validation_failed` | Request body failed FormRequest rules |
